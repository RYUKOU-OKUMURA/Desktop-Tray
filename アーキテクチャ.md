```text
│    SideRailView / TrayPanelView / TrayTabView / ItemGridView │

│  Window Orchestration (AppKit)                               │

├─────────────────────────────────────────────────────────────┤

├─────────────────────────────────────────────────────────────┤
│  Domain Layer                                                │

```

### 2.2 実行形態

Desktop Tray は MVP では **デスクトップオーバーレイ + 左レール主導のアプリ** として動作する。

メイン操作は左レールに集約し、メニューバーアイコンは必要な場合だけ最小限の補助導線として扱う。詳細なメニューバー常駐体験は MVP 以降で強化する。

```text
起動
  └─ AppDelegate / @main App
       ├─ OverlayWindowManager (全スクリーンのトレイウィンドウ管理)
       ├─ SideRailController / SideRailView (左レール管理)
       ├─ MenuBarController (任意。最小補助メニュー)
       ├─ DesktopScanner (初回スキャン)
       ├─ FileSystemWatcher (Desktop 監視)
       └─ PersistenceStore (状態復元)
```

- メインウィンドウは持たない
- 各トレイは独立した `NSPanel` としてデスクトップ上に表示
- 左レールは画面左端に固定表示し、トレイ作成・収納タブ・最小メニューを担う
- 収納時はトレイ本体を隠し、左端タブ用の小型パネルのみ表示

## 3. レイヤー設計

### 3.1 Presentation Layer

SwiftUI で UI を構成し、必要箇所だけ AppKit bridge を使う。

| コンポーネント | 役割 |
|----------------|------|
| `SideRailView` | 左レール。新規トレイ、収納タブ一覧、最小メニュー入口 |
| `TrayPanelView` | トレイ本体。ヘッダー、件数、グリッド、収納ボタン |
| `TrayItemView` | 単一ファイル/フォルダ表示。ホバー、D&D ソース/ターゲット |
| `TrayTabView` | 左端収納タブ |
| `SnapGuideOverlay` | 左端吸着ガイド |
| `SettingsView` | MVP 以降。透明度、アニメーション、表示モード |
| `ToastView` | 「あとで読むに追加しました」等のフィードバック |

Glass UI の具体的な API 選定は [技術スタック v0.1](./技術スタック.md) を参照。

### 3.2 Window Orchestration Layer

トレイ体験の核。SwiftUI だけでは不足する部分を AppKit で担う。

| コンポーネント | 役割 |
|----------------|------|
| `TrayWindowController` | 1 トレイ = 1 `NSPanel` の生成・更新・破棄 |
| `OverlayWindowManager` | 全トレイのライフサイクル、Z-order、画面外補正 |
| `SideRailController` | 左レール用パネルの生成・配置・画面外補正 |
| `MenuBarController` | 任意の `NSStatusItem`。詳細メニューは MVP 以降 |
| `DragSessionBridge` | 外部 D&D（Finder からのドロップ）受け取り |

- Mission Control / Spaces 対応は `collectionBehavior` で早期に検証
- 外部ディスプレイ切断時は `OverlayWindowManager` が frame を可視領域内へ clamp

### 3.3 Application Layer

UI 状態とユースケース実行の境界。

| コンポーネント | 役割 |
|----------------|------|
| `AppCoordinator` | 起動、復元、Desktop 監視の orchestration。表示モード切替は MVP 以降 |
| `TrayListViewModel` | 全トレイ一覧、作成/削除、全展開/全収納 |
| `TrayPanelViewModel` | 1 トレイの items、件数、収納/展開、D&D 結果反映 |

ViewModel は Domain サービスを注入して使い、**ファイルパス文字列を View に直接渡さない**（表示用 `TrayItemPresentation` へ変換）。

### 3.4 Domain Layer

ビジネスルールの中心。UI フレームワーク非依存。

| モデル / サービス | 役割 |
|-------------------|------|
| `Tray` | id, name, type, color, frame, collapseState, items/rule |
| `TrayItem` | fileURL, sortIndex, stale 状態 |
| `SmartTrayRule` | 条件種別 + パラメータ |
| `DesktopDisplayMode` | `.safe` / `.clean` |
| `TrayEngine` | 手動トレイへの追加/削除/並び替え/トレイ間移動 |
| `SmartTrayEvaluator` | ルール評価。結果は保存せず都度計算 |
| `LayoutEngine` | 収納/展開時の frame 計算、左端 snap 判定 |

**非破壊の担保**

`TrayEngine` が提供する操作は次に限定する。

```text
assignToTray(url, trayId)
removeFromTray(url, trayId)
reorder(url, trayId, newIndex)
moveBetweenTrays(url, fromTrayId, toTrayId)
updateTrayLayout(trayId, frame, collapsed)
```

ファイルシステムを変更する API は Domain に置かない。

### 3.5 Infrastructure Layer

外部世界（macOS ファイルシステム、Finder、永続化）との接点。

| コンポーネント | 役割 |
|----------------|------|
| `DesktopScanner` | `~/Desktop` 直下を列挙し `DesktopItem` 化 |
| `FileSystemWatcher` | FSEvents で Desktop 変更を検知 |
| `IconProvider` | MVP は標準アイコン。MVP 以降で Quick Look サムネ |
| `FileActionsService` | open / reveal |
| `PersistenceStore` | JSON 読み書き、schema version 管理 |
| `DesktopVisibilityController` | MVP 以降。安全/スッキリモード切替 |

## 4. 主要ユースケースとデータフロー

### 4.1 起動〜復元

```text
App Launch
  → PersistenceStore.load()
  → DesktopScanner.scanDesktop()
  → SmartTrayEvaluator.evaluate(allDesktopItems, trays)
  → OverlayWindowManager.restoreWindows(from: trays)
  → FileSystemWatcher.start()
  → Tray UI 表示 (目標: 1 秒以内)
```

### 4.2 ファイルをトレイに追加

```text
Drop on TrayPanel
  → DragSessionBridge から file URL 取得
  → TrayEngine.assignToTray(url, trayId)
  → PersistenceStore.save()
  → TrayPanelViewModel refresh
  → Toast 表示
```

**重要:** このフローに `FileManager.moveItem` は存在しない。

### 4.3 スマートトレイ表示

スマートトレイは **保存された membership リストを持たない**。

```text
Desktop items
  → SmartTrayEvaluator.matches(rule, item)
  → 条件一致 item を TrayPanelViewModel が表示用に合成
```

同一ファイルが複数トレイに表示されるのは仕様どおり。  
「未分類」は **手動トレイ未所属** かつ **他スマートルール非該当** として計算する。

### 4.4 収納 / 展開

```text
Drag tray to left edge
  → LayoutEngine.shouldSnap(frame) == true
  → SnapGuideOverlay 表示
  → Drop
  → LayoutEngine.collapsedTabFrame(...)
  → Tray.isCollapsed = true
  → PersistenceStore.save()
  → spring animation

Tab click
  → restore saved expanded frame
  → Tray.isCollapsed = false
```

## 5. 永続化設計

### 5.1 保存場所

```text
~/Library/Application Support/DesktopTray/
  ├── trays.json
  ├── settings.json
  └── schema-version.txt
```

### 5.2 保存対象

要件定義どおり、**ファイル本体ではなくメタデータのみ**。

| キー | 内容 |
|------|------|
| `trays[].id` | UUID |
| `trays[].name` | 表示名 |
| `trays[].type` | `manual` / `smart` |
| `trays[].color` | トレイ識別色 |
| `trays[].frame` | 展開時 rect |
| `trays[].isCollapsed` | 収納状態 |
| `trays[].tabIndex` | 左端タブ順 |
| `trays[].items[]` | 手動トレイのみ。`url`, `sortIndex` |
| `trays[].rule` | スマートトレイのみ |
| `settings.displayMode` | MVP は `safe` 固定。`clean` は MVP 以降 |
| `settings.appearance` | MVP 以降。透明度、角丸、アニメーション |

### 5.3 欠損ファイルの扱い

- JSON に存在する URL が Desktop 上にない → `stale` として UI 表示、クラッシュしない
- Desktop に存在するが JSON にない → スマートトレイ/未分類の評価対象
- 保存失敗時 → メモリ状態は保持、次回再試行。ユーザーへ軽い警告

### 5.4 将来移行

- `schema-version` を導入し、SQLite 移行時も JSON import path を残す
- 手動トレイ items だけ DB 化、スマートトレイは rule 定義のみ保存、という分割も可能

## 6. スマートトレイ設計

### 6.1 ルール型（MVP）

```swift
enum SmartRuleKind {
  case filenameContains(String)
  case fileExtensionIn([String])
  case createdWithinDays(Int)
  case uncategorized
}
```

### 6.2 MVP プリセット

| トレイ名 | ルール |
|----------|--------|
| スクリーンショット | ファイル名に `スクリーンショット` or `Screenshot` を含む |
| PDF | 拡張子 `.pdf` |
| 画像 | `.png`, `.jpg`, `.jpeg`, `.webp`, `.heic`, `.gif` |
| 最近追加 | 作成日 or 追加日が 7 日以内 |
| 未分類 | 手動未所属 かつ 上記スマートルール非該当 |

### 6.3 評価タイミング

- 起動時
- Desktop 変更イベント受信時
- 手動トレイ membership 更新時

評価は **バックグラウンドキュー** で行い、UI へは diff 更新を渡す。500 ファイル規模でも UI thread を塞がない。

## 7. デスクトップ表示モード

要件定義のとおり、MVP は **安全モード固定** とする。

安全モード / スッキリモードは技術的に難易度が大きく異なるため、モード切替は MVP に含めない。

### 7.1 安全モード（MVPで固定）

```text
Finder デスクトップアイコン: そのまま
Desktop Tray: 半透明トレイと左レールを上にフローティング表示
```

- 実装コスト低
- 初期値 / デフォルト
- 二重表示は許容
- Finder 設定を変更しない
- `DesktopDisplayMode.safe` として扱い、MVP ではユーザーにモード選択を出さない

### 7.2 スッキリモード（MVP以降）

```text
Finder デスクトップアイコン: 非表示
Desktop Tray: ファイル表示の主 UI
```

**実装候補**

| 方式 | 概要 | 長所 | 短所 |
|------|------|------|------|
| A. Finder 設定切替 | `CreateDesktop` 相当の設定でアイコン非表示 | 比較的確実 | ユーザー環境への介入。復元必須 |
| B. オーバーレイ完全代替 | Finder アイコンはそのまま、視覚的に隠す | 非破壊 | 完全な「片付き感」は出にくい |
| C. Desktop レベル再実装 | 壁紙上に独自アイコンレイアウト | 体験最高 | 実装/保守コスト大 |

**MVP 以降の推奨:** 方式 A を **明示 opt-in** で採用。  
`DesktopVisibilityController` が状態を記録し、アプリ終了時/モード OFF 時に必ず元へ戻す。

### 7.3 安全装置

- 初回は安全モード固定
- スッキリモード ON 前に確認ダイアログ
- メニューからいつでも安全モードへ戻せる
- 設定ファイルに「最後に変更した主体=DesktopTray」を記録

## 9. Phase 別 実装マップ

要件定義の Phase 1〜4 を、触るモジュールに対応づける。

MVP範囲は Phase 1〜3。Phase 4 以降は MVP 後の体験強化。

### Phase 1: 見た目プロトタイプ

**目的:** 触り心地の再現。実ファイル不要。

| 実装 | モジュール |
|------|------------|
| 半透明トレイ UI | `GlassBackgroundView`, `TrayPanelView` |
| 左レール | `SideRailController`, `SideRailView`, `TrayTabView` |
| ダミー item | Preview / mock data |
| ドラッグ移動 | `TrayWindowController` |
| 左端収納 | `LayoutEngine`, `TrayTabView` |
| タブ展開 | `OverlayWindowManager` + spring animation |

**完了条件:** ダミーデータで「気持ちいいトレイ」と左レールが動く。

### Phase 2: 実ファイル連携

| 実装 | モジュール |
|------|------------|
| Desktop 読込 | `DesktopScanner` |
| アイコン表示 | `IconProvider` |
| 開く / Finder 表示 | `FileActionsService` |
| 手動分類保存 | `TrayEngine`, `PersistenceStore` |
| Desktop 変更反映 | `FileSystemWatcher` |

**完了条件:** 実ファイルをトレイに入れてもパスが変わらない。

### Phase 3: スマートトレイ

| 実装 | モジュール |
|------|------------|
| 5 種プリセット | `SmartTrayRule`, `SmartTrayEvaluator` |
| Smart バッジ UI | `TrayPanelView` |
| 未分類ロジック | `SmartTrayEvaluator.uncategorized` |

**完了条件:** 同一ファイルの多重表示が仕様どおり動く。

### Phase 4: MVP以降の体験強化

| 実装 | モジュール |
|------|------------|
| スッキリモード | `DesktopVisibilityController` |
| トレイ色 | `TrayTheme` |
| Quick Look サムネイル | `IconProvider` + Quick Look |
| 設定画面 | `SettingsView` |
| メニューバー詳細メニュー | `MenuBarController` |
| Login Item | `ServiceManagement` / SMAppService |
| スマートトレイ条件編集 | `SmartTrayRuleEditor`, `SmartTrayEvaluator` |

## 10. 主要リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| デスクトップ上ウィンドウが他アプリと競合 | トレイが埋もれる/操作不能 | `level` と `nonactivatingPanel` を早期検証 |
| Spaces / 外部ディスプレイで位置狂い | トレイ消失 | `OverlayWindowManager.clampToVisibleFrames()` |
| スッキリモードの Finder 介入 | ユーザー不安 | opt-in、ワンクリック復元、安全モードデフォルト |
| ファイル削除・改名 | stale item | `DesktopScanner` diff + graceful UI |
| Sandbox 化 | Desktop アクセス失敗 | Phase 4 で bookmark 対応 |
| 過剰アニメーション | 操作遅延 | Reduce Motion 対応、60fps 優先 |
| スマートトレイ再計算コスト | CPU 使用 | 背景 queue + 結果キャッシュ（短 TTL） |

## 11. テスト方針

### 11.1 単体テスト

- `SmartTrayEvaluator` ルール判定
- `LayoutEngine` snap / collapse frame
- `TrayEngine` の非破壊操作
- `PersistenceStore` encode/decode

### 11.2 結合テスト

- Desktop 変更 → UI 更新
- 収納 → 再起動 → 復元
- stale file 表示

### 11.3 手動 QA（MVP 必須）

- ライト/ダーク切替
- 外部ディスプレイ抜き差し
- Mission Control 表示
- 100 / 500 ファイルでのスクロール性能
- 安全モードでの Finder 共存
