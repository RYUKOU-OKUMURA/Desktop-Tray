# Desktop Tray 技術スタック v0.1

> 関連ドキュメント: [要件定義 v0.1](./要件定義.md) / [アーキテクチャ v0.1](./アーキテクチャ.md)

## 1. ドキュメントの目的

本書は、要件定義 v0.1 に基づく Desktop Tray の **技術選定** を整理する。

- 採用する言語・フレームワーク・macOS API
- 採用しない選択肢とその理由
- UI / 永続化 / 監視 / 配布に関する技術方針

システム構成・モジュール分割・データフローは [アーキテクチャ v0.1](./アーキテクチャ.md) を参照。

---

## 2. 推奨スタック

| 区分 | 選定 | 理由 |
|------|------|------|
| 言語 | **Swift 5.10+** | macOS API との親和性、型安全、将来の App Store 配布 |
| UI | **SwiftUI + AppKit ハイブリッド** | SwiftUI でトレイ UI、AppKit でウィンドウ制御・Glass 素材 |
| アーキテクチャパターン | **MVVM + ドメインサービス** | UI 状態とファイルシステム操作を分離しやすい |
| 永続化 | **JSON (Codable)** | MVP 要件どおり。将来 SQLite へ移行可能 |
| ファイル監視 | **FSEvents** | デスクトップ変更のリアルタイム反映 |
| アイコン | **NSWorkspace 標準アイコン** | MVP は軽量に表示。Quick Look サムネイルは MVP 以降 |
| ビルド | **Xcode + Swift Package Manager** | 標準的な Mac アプリ開発フロー |
| 最小 OS | **macOS 14 (Sonoma)** | `NSVisualEffectView`、最新 material、SwiftUI 安定性 |
| 推奨 OS | **macOS 15 (Sequoia) 以降** | Liquid Glass 系の最新デザイン言語に近い表現 |

---

## 3. 採用しない選択肢

| 候補 | 不採用理由 |
|------|------------|
| Electron / Tauri | 半透明・ブラー・デスクトップレベルウィンドウの再現コストが高い |
| React Native for macOS | Finder 連携、ファイル監視、メニューバー常駐の実装が非効率 |
| 純 SwiftUI のみ | デスクトップ上フローティングウィンドウ、D&D、メニューバーは AppKit 依存が強い |
| WebView ベース UI | Mac らしい触り心地・アニメーション・vibrancy を損なう |

---

## 4. 設計原則

要件定義のプロダクト方針を、技術原則に対応づける。

| プロダクト方針 | 技術原則 |
|----------------|----------|
| 非破壊 | ファイル操作 API は `open` / `reveal` のみ。`move` / `remove` は UI 層から呼ばない |
| 見た目がスッキリ | 安全モード固定で Finder には介入せず、デスクトップレベルの透明ウィンドウ + 左レール + system material で視界を整理 |
| 操作がシンプル | ドメインモデルを「Tray = 参照集合」に限定 |
| Mac らしい | AppKit の system material / NSWorkspace / 標準 D&D を優先 |

---

## 5. 主要 macOS API

| 用途 | API / フレームワーク |
|------|----------------------|
| 半透明背景 | `NSVisualEffectView` + `NSViewRepresentable` |
| フローティングウィンドウ | `NSPanel` |
| メニューバー補助 | `NSStatusItem`。MVP では任意の最小メニュー、詳細メニューは MVP 以降 |
| ファイル列挙 | `FileManager.contentsOfDirectory` |
| ファイル監視 | FSEvents |
| アイコン取得 | `NSWorkspace.shared.icon(forFile:)` |
| ファイルを開く | `NSWorkspace.shared.open(_:)` |
| Finder で表示 | `NSWorkspace.shared.activateFileViewerSelecting(_:)` |
| 外部 D&D | `NSDraggingDestination` / `DragSessionBridge` |
| 起動時自動開始 | `SMAppService` / `ServiceManagement`。MVP 以降 |
| アクセシビリティ | `NSWorkspace.shared.accessibilityDisplayShouldReduceMotion` |

## 6. UI 技術方針

### 6.1 Glass / Liquid Glass

- 背景: `NSVisualEffectView` を SwiftUI に埋め込む
- Material: `.hudWindow` または `.sidebar` をベースに、透明度設定で調整
- 角丸・枠線・影: SwiftUI modifier で上乗せ
- ライト/ダーク: `@Environment(\.colorScheme)` + system color 追従

### 6.2 ウィンドウ設定（MVP）

```swift
panel.styleMask = [.nonactivatingPanel, .borderless, .resizable]
panel.level = .floating                    // Phase 1。必要なら .statusBar 等で調整
panel.collectionBehavior = [
  .canJoinAllSpaces,
  .stationary,
  .ignoresCycle,
  .fullScreenAuxiliary
]
panel.isOpaque = false
panel.backgroundColor = .clear
panel.hasShadow = true
```

Phase 1 では「デスクトップの上に載る半透明パネル」を最優先する。

### 6.3 アニメーション

| 要件 | 実装 |
|------|------|
| トレイドラッグ追従 | `NSPanel.setFrame(..., display: true)` |
| 左端吸着 | `LayoutEngine` + `SnapGuideOverlay` |
| 収納/展開 | `NSAnimationContext` または SwiftUI `.spring()` |
| 追加フェード | item `transition(.opacity)` |
| ホバー浮き | `scaleEffect(1.03)` + shadow |

`Reduce Motion` 有効時は spring を短縮または無効化する。

### 6.4 パフォーマンス

- アイコン: まず `NSWorkspace` 標準アイコン
- サムネイル: MVP 以降に `QLThumbnailGenerator` を低優先度 queue で生成
- item グリッド: `LazyVGrid` + item ID 安定化
- 500 件規模: スクロール + 遅延読込

### 6.5 アクセシビリティ

- MVPではライト/ダークモードと Reduce Motion などのシステム設定に追従
- 透明度設定で material opacity を上げる調整 UI は MVP 以降
- VoiceOver label: トレイ名、件数、ファイル名
- 大きめ文字でもレイアウトが崩れないよう Dynamic Type を考慮

## 7. 永続化・データ形式

| 項目 | 選定 |
|------|------|
| 形式 | JSON (Codable) |
| 保存場所 | `~/Library/Application Support/DesktopTray/` |
| 主要ファイル | `trays.json`, `settings.json`, `schema-version.txt` |
| 将来 | SQLite 移行可。schema version で互換管理 |

保存するのはファイル本体ではなく **メタデータのみ**。詳細スキーマは [アーキテクチャ v0.1](./アーキテクチャ.md) の永続化設計を参照。

## 8. 権限・セキュリティ・配布

### 8.1 必要権限

| 権限 | 用途 |
|------|------|
| Desktop フォルダアクセス | スキャン、監視、アイコン取得 |
| ファイル開封 | ダブルクリックで関連アプリ起動 |
| Finder 表示 | `activateFileViewerSelecting` |

### 8.2 サンドボックス方針

| 配布形態 | 方針 |
|----------|------|
| 開発初期（Developer ID 直配布） | App Sandbox OFF でも可。Phase 1 を優先 |
| Mac App Store 想定 | App Sandbox ON + Security-Scoped Bookmark + Desktop Access entitlement |

MVP 実装順:

1. 非サンドボックスで核心 UX を完成
2. 永続化 URL を bookmark 化
3. Sandbox 対応を Phase 4 以降で追加

### 8.3 プライバシー

- ネットワーク通信なし（MVP）
- テレメトリなし
- ファイル内容を読み込まない（メタデータ + アイコンのみ）
