# Macデスクトップ仮想トレイアプリ 要件定義 v0.1

## 1. プロダクト概要

### 仮称

**Desktop Tray**  
または日本語なら **デスクトップトレイ**

### 一言でいうと

Macのデスクトップ上にあるファイルやフォルダを、**実体を動かさずに、半透明の美しい仮想トレイで整理できるアプリ**。

### コアコンセプト

> ファイルは動かさない。  
> デスクトップの見え方だけを整理する。  
> 必要なものはトレイにまとめ、不要な時は画面端にしまう。

### 重要な価値

このアプリの価値は、ファイル管理の効率化よりも **視界の整理** と **気持ちよさ**。

ユーザーが求めているのは、

```text
デスクトップが散らかって見えない
後で読む資料を一時的にまとめたい
作業中の画像や素材をまとめたい
プロジェクトフォルダの場所は変えたくない
邪魔なものはサッと画面端にしまいたい
Macっぽくて美しいUIで使いたい
```

という体験。

---

## 2. プロダクト方針

### 最重要方針

#### 1. 非破壊

ファイルやフォルダの実体は移動しない。

```text
~/Desktop/sample.pdf
```

にあるファイルは、トレイに入れてもその場所のまま。

アプリ側では、

```text
あとで読む トレイ
  - ~/Desktop/sample.pdf への参照
```

というメタデータだけを持つ。

#### 2. 見た目がスッキリ

ユーザーがアプリを使った瞬間に、

```text
あ、デスクトップが片付いた
```

と感じることを最優先にする。

#### 3. 操作がシンプル

複雑なファイル管理アプリにはしない。

初期体験は、

```text
トレイを作る
ファイルを入れる
トレイをしまう
必要な時に開く
```

だけで成立させる。

#### 4. Macらしい

macOS標準の質感、角丸、余白、アニメーション、ブラー、ライト/ダークモード追従を重視する。

Appleの現在のデザイン言語では、半透明で周囲に反応する素材は **Liquid Glass** として説明されており、周囲を反射・屈折し、ライト/ダーク環境にも適応するものとして位置づけられている。macOSネイティブ実装では `NSVisualEffectView` やシステムマテリアル/バイブランシーを使う方向が自然。（[Apple Newsroom][1]）

## 4. MVPのゴール

### MVPで実現すること

MVPの目的は、**Finder のデスクトップ表示には介入せず、安全に「美しい仮想トレイ体験」を成立させること**。

MVPは次の範囲に切る。

```text
安全モード固定で動く
Finder のデスクトップアイコンは隠さない
デスクトップ上のファイルを読み込む
仮想トレイにまとめる
トレイを半透明パネルで表示する
ファイルを移動せずに分類できる
左レールからトレイを作成・収納・展開できる
トレイを左端に収納できる
必要な時にタブから展開できる
最低限のスマートトレイで自動分類できる
```

MVPで重視する順番は、

```text
1. 安全モード固定
2. 美しいトレイ体験
3. 実ファイル非破壊
4. 左レール
5. 最低限スマートトレイ
```

スッキリモード、詳細設定、スマートトレイ条件編集、トレイ色変更、サムネイル強化などは MVP 以降で実装する。

## 5. MVP機能要件

### 5.1 デスクトップファイル読み込み

#### 概要

アプリ起動時に、ユーザーのデスクトップ上のファイル・フォルダを取得する。

#### 対象

```text
~/Desktop
```

の直下にあるファイル・フォルダ。

#### MVP対象外

デスクトップ配下のフォルダ内部まではスキャンしない。

```text
~/Desktop/project/
  src/
  package.json
```

この場合、MVPでは `project` フォルダだけを1アイテムとして扱う。

---

### 5.2 仮想トレイ

#### 概要

ファイルやフォルダをまとめる単位。

ただし、Finder上の実フォルダではない。

#### 呼び方

UI上では **Folder** ではなく **Tray** と呼ぶ。

日本語UIなら、

```text
トレイ
置き場
まとまり
```

が候補。

個人的には **トレイ** が一番いい。「フォルダ」と呼ぶと、ユーザーが実際のディレクトリだと誤解する可能性がある。

#### トレイに入れる操作の意味

```text
ファイルをトレイに入れる
= ファイルを移動するのではなく、トレイに登録する
```

#### トレイから外す操作の意味

```text
トレイから外す
= アプリ上の分類から外すだけ
= ファイルは削除されない
= ファイルは移動されない
```

---

### 5.3 初期トレイ

初回起動時には、以下のトレイを用意する。

```text
あとで読む
素材
作業中
一時保存
その他
```

`スクリーンショット` はスマートトレイとして提供するため、手動の初期トレイには含めない。

英語UIなら、

```text
Read Later
Assets
In Progress
Temp
Other
```

MVPでは日本語UI優先でよい。

---

### 5.4 トレイ表示

#### 基本UI

トレイは、デスクトップ上に浮かぶ半透明パネルとして表示する。

#### トレイの見た目

```text
半透明背景
背景ブラー
角丸
薄い枠線
柔らかい影
タイトル
件数バッジ
アイコングリッド
右上の収納ボタン
```

#### レイアウト例

```text
┌──────────────────────────────┐
│ あとで読む        7        ▸ │
│                              │
│  [PDF]   [PDF]   [PNG]       │
│  doc     spec    image       │
│                              │
│  [ZIP]   [KEY]   [folder]    │
│  data    slide   project     │
└──────────────────────────────┘
```

#### 件数バッジ

トレイタイトルの近くに、そのトレイ内のアイテム数を表示する。

```text
あとで読む  7
素材        12
その他      4
```

## 6. UI / ビジュアル要件

### 6.1 デザインキーワード

このアプリのUIは以下を目指す。

```text
Glass
Liquid Glass
macOSらしい
軽い
静か
邪魔しない
綺麗
触って気持ちいい
```

いわゆる「グラスUI」は、一般的には Glassmorphism と呼ばれることが多い。ただ、Macアプリとして作るなら、表現上は **Liquid Glass / macOS material / vibrancy** に寄せた方がよい。AppleはLiquid Glassを、周囲のコンテンツやコンテキストに応じて変化し、アプリの操作要素・ナビゲーション・アイコン・ウィジェットなどに活用される新しい素材として説明している。（[Apple Newsroom][1]）

### 6.2 トレイの質感

#### 通常状態

```text
背景: 半透明
ブラー: あり
角丸: 大きめ
枠線: 1px程度
影: 控えめ
透明度: 背景がうっすら見える程度
```

#### ホバー状態

```text
枠線が少し明るくなる
タイトルバーがわずかに強調される
カーソルが当たったアイテムだけ軽く浮く
```

#### ドラッグ状態

```text
トレイ全体が少し拡大
影が少し強くなる
左端に近づくと吸着ガイドが出る
```

#### 収納状態

```text
左端に細い縦タブとして表示
タブにはトレイ名の先頭文字または短縮名を表示
色でトレイを識別
件数を小さく表示してもよい
```

## 7. 操作要件

### 7.1 トレイ作成

ユーザーは新しいトレイを作成できる。

#### 操作

```text
左レールの + ボタン
```

#### 入力項目

MVPでは、トレイ名だけでよい。

```text
トレイ名
```

MVP以降に、以下を追加する。

```text
色
スマートトレイにするかどうか
```

デスクトップ上で右クリックして新規トレイを作る操作も MVP 以降でよい。

---

### 7.2 ファイルをトレイに入れる

#### 操作

デスクトップ上のファイルをトレイにドラッグ&ドロップする。

#### 挙動

```text
ファイル実体は移動しない
アプリ内の分類情報だけが更新される
トレイの件数が増える
```

#### 表示

ドロップ時に小さく、

```text
「あとで読む」に追加しました
```

のようなフィードバックを出す。

---

### 7.3 トレイ間移動

#### 操作

トレイ内のアイテムを別トレイへドラッグする。

#### 挙動

```text
所属トレイだけ変更
ファイル実体は移動しない
```

---

### 7.4 並び替え

#### 操作

トレイ内のアイテムをドラッグして並び替える。

#### 挙動

```text
表示順だけ保存
ファイルシステムには影響しない
```

---

### 7.5 ファイルを開く

#### 操作

アイテムをダブルクリック。

#### 挙動

macOS標準の関連アプリで開く。

例：

```text
PDF -> Preview
画像 -> Preview / 既定アプリ
Keynote -> Keynote
フォルダ -> Finder
```

---

### 7.6 右クリックメニュー

MVPでは最低限これだけでよい。

```text
開く
Finderで表示
トレイから外す
```

将来的には以下も追加。

```text
名前を変更...
ゴミ箱に入れる...
情報を見る
クイックルック
別のトレイへ移動
```

ただし、危険操作は明確に分ける。

```text
トレイから外す
```

と

```text
ゴミ箱に入れる
```

は絶対に別物として扱う。

---

## 8. 収納 / 展開要件

### 8.1 左端収納

#### 概要

トレイを画面左端へドラッグすると、縦タブとして収納される。

#### 操作

```text
トレイを左端へドラッグ
画面端に近づく
吸着ガイドが表示される
ドロップするとタブ化
```

#### 挙動

```text
展開時の位置とサイズを保存
トレイ本体は非表示
左端にタブだけ残る
```

---

### 8.2 タブから展開

#### 操作

左端のタブをクリック。

#### 挙動

```text
元のサイズ・位置でトレイが展開される
タブは非表示または展開中表示に変わる
```

#### アニメーション

動画の魅力を再現するなら、ここはかなり重要。

```text
タブからスッと出てくる
少しだけ弾性がある
速すぎず遅すぎない
Macっぽく自然
```

---

### 8.3 収納位置

MVPでは左端のみ。

将来的には、

```text
右端
上端
下端
```

も対応できるが、MVPではやらない方がいい。

UIをシンプルに保つため、最初は **左端固定** がいい。

---

## 9. スマートトレイ要件

これはかなり魅力的なので、MVPに入れてよい。ただし、最初から複雑にしすぎない。

### 9.1 スマートトレイとは

条件に一致するデスクトップ上のファイルを、自動的に集める仮想トレイ。

ファイルは移動しない。

#### 例

```text
スクリーンショット
PDF
画像
最近追加
動画
ZIP
未分類
名前に「2026」を含む
```

---

### 9.2 MVPで入れるスマートトレイ

最初に入れるなら、この5つが良い。

```text
スクリーンショット
PDF
画像
最近追加
未分類
```

#### スクリーンショット

条件例：

```text
ファイル名に「スクリーンショット」
または
ファイル名に "Screenshot"
または
画像ファイルかつ作成元がスクショっぽい
```

#### PDF

条件例：

```text
拡張子が .pdf
```

#### 画像

条件例：

```text
拡張子が .png
.jpg
.jpeg
.webp
.heic
.gif
```

#### 最近追加

条件例：

```text
作成日または追加日が過去7日以内
```

#### 未分類

条件例：

```text
どの手動トレイにも入っていない
どのスマートトレイにも該当しない
```

ただし、未分類は扱いに注意。スマートトレイの仕様によっては、同じファイルが複数トレイに出る可能性がある。

---

### 9.3 スマートトレイの表示ルール

スマートトレイは、通常トレイと見た目を少し変える。

#### 通常トレイ

```text
ユーザーが手動で入れたもの
```

#### スマートトレイ

```text
条件により自動で表示されるもの
```

#### UI上の違い

```text
タイトル横に小さな魔法アイコン
または
「Smart」バッジ
または
薄い点線の枠
```

ただし、MVPではやりすぎない。

例：

```text
PDF  12  Smart
```

くらいで十分。

---

### 9.4 スマートトレイの編集

MVPではプリセットのみでよい。

つまり、

```text
スマートトレイの条件を自由に作る
```

まではやらない。

MVPでは `条件を編集...` のような操作 UI は表示しない。表示するとしても無効状態にし、ユーザーが条件を変更できるようにはしない。

MVP後に、

```text
拡張子
ファイル名
作成日
更新日
サイズ
タグ
種類
```

で条件を作れるようにする。

---

### 9.5 スマートトレイと手動トレイの関係

ここは大事。

#### 方針

同じファイルは、手動トレイにもスマートトレイにも表示されてよい。

例：

```text
design.png
```

がある場合、

```text
素材
画像
最近追加
```

の3つに表示されてもよい。

なぜなら、スマートトレイは「実体の所属」ではなく「条件ビュー」だから。

ただし、ユーザーが混乱しないようにする。

#### 表示上の説明

スマートトレイ内には小さく、

```text
条件に一致するファイルを表示しています
```

と説明してもよい。

## 10. デスクトップの見え方に関する重要仕様

ここが一番プロダクト体験に影響する。

### 課題

ファイルを動かさない場合、Finderのデスクトップアイコンと、アプリのトレイ内アイコンが二重に見える可能性がある。

つまり、

```text
Finderのデスクトップに sample.pdf が見える
トレイ内にも sample.pdf が見える
```

となる。

これだと「見た目がスッキリ」という価値が弱くなる。

---

### MVP仕様

MVPでは **安全モード固定** とする。

アプリを有効化すると、Finder のデスクトップアイコンはそのままに、Desktop Tray の半透明トレイと左レールを上に重ねて表示する。

MVPでは安全モード / スッキリモードの切り替え UI は出さない。スッキリモードは MVP 以降の体験強化として実装する。

---

### 10.1 安全モード

#### 概要

Finderのデスクトップアイコンはそのまま表示する。

アプリは上にトレイを重ねるだけ。

#### メリット

安全。Finderの挙動を変えない。実装が簡単。

#### デメリット

見た目のスッキリ感は弱い。

#### MVPでの扱い

初期版ではこのモードだけを実装する。

```text
Finderのデスクトップアイコンは隠さない
Finder設定を変更しない
実ファイルを移動しない
トレイと左レールだけを重ねる
```

---

### 10.2 スッキリモード

#### 概要

Finderのデスクトップアイコンを隠し、アプリのトレイUIでデスクトップ上のファイルを表示する。

#### メリット

動画に近い体験になる。デスクトップが本当に片付いて見える。プロダクトの魅力が一番伝わる。

#### デメリット

Finderのデスクトップ操作をアプリ側で代替する必要がある。

#### 導入時に代替すべき操作

```text
開く
Finderで表示
トレイから外す
ドラッグ&ドロップ
```

#### その後に代替する操作

```text
クイックルック
名前変更
ゴミ箱に入れる
情報を見る
タグ編集
共有
```

---

### 10.3 MVP以降の導入方針

MVP以降も、初回起動時は **安全モード** にする。

その後、オンボーディングで、

```text
デスクトップをもっとスッキリ表示しますか？
```

と聞いて、ユーザーが明示的に許可したらスッキリモードを有効にする。

スッキリモードを入れる時は、必ずワンクリックで安全モードへ戻せるようにする。

これが一番信頼を失わない。

## 11. データ保存要件

### 保存するもの

アプリは、ファイルそのものではなく、以下のメタデータを保存する。

```text
トレイ一覧
トレイ名
トレイ色
トレイ位置
トレイサイズ
収納状態
収納タブ位置
手動で登録されたファイルURL
スマートトレイ設定
アイテムの表示順
```

### 保存形式

MVPではJSONでよい。

例：

```json
{
  "trays": [
    {
      "id": "tray-read-later",
      "name": "あとで読む",
      "type": "manual",
      "color": "blue",
      "frame": {
        "x": 120,
        "y": 80,
        "width": 420,
        "height": 300
      },
      "isCollapsed": false,
      "items": [
        {
          "url": "/Users/user/Desktop/spec.pdf",
          "sortIndex": 0
        }
      ]
    },
    {
      "id": "tray-screenshots",
      "name": "スクリーンショット",
      "type": "smart",
      "rule": {
        "kind": "filename_contains",
        "value": "スクリーンショット"
      },
      "isCollapsed": true
    }
  ]
}
```

将来的にはSQLiteでもよいが、MVPはJSONで十分。

## 12. 権限 / プライバシー要件

### 基本方針

このアプリはローカル完結が望ましい。

#### 原則

```text
ファイル内容を外部送信しない
クラウド同期しない
AI分類を入れる場合も初期はローカルルールベース
ユーザーのデスクトップ外は勝手にスキャンしない
```

### 必要な権限

```text
デスクトップフォルダへのアクセス
ファイルを開く権限
Finderで表示する権限
```

Mac App Store配布を想定する場合、サンドボックスやファイルアクセス設計が重要になる。

---

## 13. 非機能要件

### 13.1 パフォーマンス

#### 起動

```text
アプリ起動後、1秒以内にトレイUIを表示
```

#### デスクトップスキャン

```text
100ファイル程度なら即時表示
500ファイル程度でも操作が重くならない
```

#### サムネイル

MVPでは標準アイコンを基本にする。

画像やPDFの Quick Look サムネイル生成は MVP 以降の体験強化として遅延読み込みで追加する。

```text
まず標準アイコンを表示
MVP以降にサムネイルへ差し替え
```

---

### 13.2 安定性

```text
ファイルが削除されてもアプリが落ちない
ファイル名が変わっても壊れない
外部ディスプレイを外してもトレイが画面外に残らない
Mission Control / Spaces で破綻しない
```

---

### 13.3 アクセシビリティ

半透明UIは綺麗だが、読みづらくなりやすい。

最低限、

```text
文字コントラストを確保
ライト/ダークモード対応
大きめ文字でも崩れない
```

を入れる。

MVPでは独立した設定画面を作らないため、透明度やアニメーションのユーザー設定は MVP 以降で追加する。MVPでは macOS のライト/ダークモードと Reduce Motion などのシステム設定にできる範囲で追従する。

---

## 14. MVPでやらないこと

初期版では、以下はやらない。

```text
実ファイルの自動移動
自動リネーム
自動削除
重複削除
フォルダ内部の再整理
Finderデスクトップアイコンの非表示
安全モード / スッキリモードの切り替え
スマートトレイ条件編集
トレイ色変更
独立した詳細設定画面
起動時自動開始
Quick Look サムネイル生成
クラウド同期
チーム共有
AIによる高度な分類
複雑なルールビルダー
右端/上端/下端への収納
```

特に、

```text
勝手にファイルを動かす
勝手に名前を変える
勝手に削除する
```

は絶対に避ける。

このアプリの信頼は **非破壊** にある。

## 15. MVPの画面構成

### 15.1 デスクトップオーバーレイ

メイン画面。ユーザーのデスクトップ上にトレイが表示される。

表示要素：

```text
左レール
トレイパネル
トレイ内アイテム
収納タブ
吸着ガイド
```

---

### 15.2 左レール

MVPの主操作面。画面左端に常時表示し、完成イメージのようにトレイの収納状態を一覧できるようにする。

表示要素：

```text
新規トレイ作成ボタン
収納中トレイの縦タブ
各タブの色
各タブの短縮名またはアイコン
設定/メニュー入口の歯車ボタン
```

MVPでは歯車ボタンの中身は最小限でよい。

```text
すべて展開
すべて収納
終了
```

詳細設定は MVP 以降で実装する。

---

### 15.3 メニューバーアイコン

MVPでは主操作を左レールに寄せる。メニューバーアイコンは必須の主導線にしない。

常駐アプリとして必要な場合のみ、最小限の補助メニューを置く。

メニュー項目：

```text
表示/非表示
終了
```

以下は MVP 以降でよい。

```text
新しいトレイ
スマートトレイを追加
すべて展開
すべて収納
安全モード / スッキリモード
設定
```

---

### 15.4 設定画面

MVPでは独立した設定画面を作らない。

設定画面は MVP 以降で実装する。

設定項目：

```text
起動時に自動開始
デスクトップ整理ビューを有効化
透明度
トレイの角丸
アニメーション
スマートトレイ
ライト/ダーク追従
```

## 18. 受け入れ条件

MVP完成の判断基準はこれ。

```text
アプリを起動すると安全モード固定で動く
Finder のデスクトップアイコンは隠れない
アプリを起動するとデスクトップ上に半透明トレイと左レールが表示される
左レールの + ボタンからトレイを新規作成できる
デスクトップ上のファイルをトレイに登録できる
登録してもファイルの場所は変わらない
トレイ内のファイルをダブルクリックで開ける
右クリックでFinder表示できる
トレイをドラッグで動かせる
左端にドラッグすると収納される
左レールの収納タブをクリックすると展開される
トレイの位置・サイズ・収納状態が再起動後も復元される
PDF / 画像 / スクリーンショット / 最近追加 / 未分類 のスマートトレイが動く
UIが半透明でMacらしく見える
ライト/ダークモードで破綻しない
```

## 20. 最終的なプロダクト定義

このアプリはこう定義するのが一番よさそう。

> **Desktop Trayは、Macのデスクトップ上にあるファイルやフォルダを、実体を動かさずに視覚的に整理できる仮想トレイアプリです。後で読む資料、作業中の素材、一時保存ファイル、スクリーンショットなどを目的別にまとめ、使わない時は画面端に収納できます。Liquid Glass風の半透明UIで、デスクトップを美しく、シンプルに、非破壊で整えます。**

MVPの勝ち筋はかなり明確。**最初に作るべきものは、賢い整理アプリではなく、触って気持ちいい"デスクトップの美しいトレイ"**。スマートトレイは強いけど、あくまで主役は「見た目がスッキリする」「Macっぽくて気持ちいい」の方。
